# AGENTS.md — cloud-del-norte-website

## what this repo is

cloud-del-norte-website is the React + Vite SPA for the Cloud Del Norte AWS User Group community site. three subdomains: clouddelnorte.org (main public site), awsug.clouddelnorte.org (authenticated members area), auth.clouddelnorte.org (cognito hosted UI login). built on AWS Cloudscape Design System v3. authenticated via Amazon Cognito (authorization code grant). deployed via manual script (operational norm) or Woodpecker CI (partial recovery, not yet validated end-to-end).

the repo is part of the heraldstack haunting — a multi-agent orchestration system. agents are dispatched into this repo with scoped tasks. the .kiro/ directory contains the haunting overlay (steering docs, agent definitions, mcp config). HANDOFF.md is the accumulated session backlog owned by poltergeist-harald-cdn-product-owner.

## quick start

```bash
npm ci                                    # install dependencies
npm run dev                               # vite dev server (localhost:5173)
npm run build                             # production build (all 3 subdomain targets)
npx vitest run                            # unit/integration tests
npx @biomejs/biome ci .                   # lint + format check (must pass before commit)
./scripts/deploy-manual.sh main           # deploy main subdomain
./scripts/deploy-manual.sh auth           # deploy auth subdomain
./scripts/deploy-manual.sh awsug          # deploy awsug subdomain
./scripts/deploy-manual.sh main --dry-run # preview what would happen without deploying
```

build outputs three targets: lib/ (main), lib-auth/ (auth), lib-awsug/ (awsug). each maps to its own S3 bucket and CloudFront distribution.

## conventions

cloudscape overrides require !important on every property plus hashed-class substring selectors scoped inside container classes. never apply cloudscape color overrides globally. modal buttons need a different selector chain than page-level buttons. see .kiro/steering/cloudscape-overrides.md for the full pattern.

manual deploy script is the operational norm while woodpecker is partially recovered. always run biome ci + npm run build locally before deploying. verify with curl last-modified checks post-deploy. see .kiro/steering/deployment.md for syntax, distribution IDs, and the verification pattern.

diagnostic-first beats panic-iteration. if two fix attempts fail on the same symptom, stop iterating and instrument instead. direct SDK invoke, curl against deployed bundles, or playwright diagnostic scripts surface root cause faster than guessing. see docs/runbooks/diagnostic-first.md.

single atomic commit per logical phase. one feature = one commit. do not split a fix across commits or bundle unrelated changes. do not combine unrelated features into one commit.

public/data/feeds.json is build-time noise generated by npm run build. always `git restore public/data/feeds.json` after building. do not commit it.

cloudscape is the only permitted UI library. no MUI, no Tailwind, no other component libraries. all UI components come from @cloudscape-design/*.

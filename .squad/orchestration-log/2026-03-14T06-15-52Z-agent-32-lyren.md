## Task Summary

SideNavigation component appeared at 0px height on desktop despite content present in DOM. UX audit revealed missing `navigationOpen` and `onNavigationChange` props in AppLayout. Implemented proper state management using Cloudscape spacing standards.

---

## Root Cause Analysis

### The Bug
- SideNavigation drawer height: 0px on desktop (viewport ≥768px)
- Navigation items required scrolling; only "User Group Home" visible
- Significant white space below visible item

### Why It Happened
- `AppLayout` was missing explicit `navigationOpen` and `onNavigationChange` props
- Cloudscape defaults navigation to **closed** on all viewports when state is unmanaged
- Navigation container received `height: 0px` from CSS Grid layout when closed
- Grid template rows: `0px 0px ...` due to closed state

---

## Changes Made

### File: `src/layouts/shell/index.tsx`

**Added state management:**

```tsx
// Determine if desktop (≥768px) for responsive default
const isDesktop = typeof window !== 'undefined' && window.innerWidth >= 768;
const [navOpen, setNavOpen] = useState(isDesktop);

<AppLayout
  navigationOpen={navOpen}
  onNavigationChange={(event) => setNavOpen(event.detail.open)}
  // ... other props
/>
```

**Key implementation details:**
- State initialized per viewport width (responsive)
- Event handler wired to toggle button
- No custom CSS height overrides (let Cloudscape handle grid layout)
- Compatible with all theme/locale combinations

### Standard Rules Established

1. **Always use `navigationOpen` prop** when using AppLayout with navigation drawer
2. **Always wire `onNavigationChange`** to handle toggle events
3. **Default to open on desktop (≥768px), closed on mobile** per Cloudscape responsive patterns
4. **Never rely on Cloudscape's default navigation state** — it defaults to closed
